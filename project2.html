Advanced Data Cleaning of Credit Data: Turning a Messy Dataset into Real Insights

In today’s world, credit scoring is central to everything from getting a loan to renting a house. But behind the scenes, credit data is often messy, inconsistent, and incomplete. For this project, the dataset included both categorical and continuous features typical of credit scoring problems — but the data was far from clean.

Challenges We Faced

  • Incorrect entries (e.g., SSNs like “#F%$D@*&8”)
  • Inconsistent values (e.g., same customer, different occupation every month)
  • Missing values everywhere

Our Cleaning Strategy

Before modeling, we tackled the mess head-on with a structured cleaning process:

Incorrect Entries:
  • Replaced garbage entries in sensitive fields like SSNs with None
  • Removed spaces and special characters
  • Reformatted compound fields (e.g., Type_of_Loan) into structured categories
  • Ensured proper data types for all fields
  • Standardized placeholders like “_” and “NM” to None for imputation
Inconsistent Values:
  • Columns like Occupation were stabilized by using the mode per customer
  • Used a hybrid fill strategy: forward-fill (Jan–on) and backward-fill (Aug–back)
  • In columns like Monthly_Balance, we applied KNN imputation for smarter estimation

Handling Imbalanced Target Variable

The credit score column was highly imbalanced — with class "2" dominating. We corrected this using SMOTE (Synthetic Minority Oversampling Technique), which synthetically balances classes for fairer learning.

Modeling & Fairness

We evaluated multiple algorithms:

  • Logistic Regression
  • Random Forest
  • XGBoost
  • Gradient Boosting

Initial models scored ~98% accuracy — but they were misleading. Upon analyzing feature importance, we found biases:

  • City and Age were dominating predictions
  • This raised concerns about fairness and discrimination

After removing these biased features, accuracy dropped to 76–81%, but models became more ethical and generalizable.

Conclusion

This project shows that data cleaning is not just a technical task — it’s the foundation of trustworthy analytics. High model accuracy means nothing without fair and clean data. Our biggest takeaway: Good analytics starts with good data.
